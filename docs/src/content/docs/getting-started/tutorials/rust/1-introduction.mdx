Welcome to the **Notes CLI with Nhost and Rust** series! Over five parts you'll
build a real command-line notes manager on top of the Nhost Rust SDK that
exercises every core Nhost product: authentication, the GraphQL database,
file storage, and serverless functions.

## About This Tutorial Series

This series is divided into **5 parts**, each focusing on one Nhost product. By
the end you'll have a `notes` CLI that can:

- **Authenticate** — sign up, sign in, and stay signed in across runs with a
  persisted session (`login` / `logout` / `whoami`)
- **Manage notes** — create, list, edit, tag, and organise notes into notebooks
  over GraphQL, with per-user permissions enforced by the database
- **Attach files** — upload attachments to Nhost Storage and download them again
- **Export & share** — call a serverless function to export notes and share a
  note with another user

:::note
This is **Part 1** in the series. This part creates your Nhost project and
scaffolds the CLI so later parts can focus on one product at a time.
:::

## What You'll Learn

- Configuring an Nhost client from Rust and reusing it across commands
- Persisting a user session to disk so the CLI stays signed in
- Running GraphQL queries and mutations with JSON variables
- Designing owner-scoped row permissions in Hasura
- Uploading and downloading files with Nhost Storage
- Calling a serverless function with the caller's credentials

## Prerequisites

- A recent **Rust toolchain** (`rustup`, stable)
- Basic knowledge of Rust and `async`/`await`
- An [Nhost account](https://app.nhost.io/) (the free tier is enough)

## Scaffold the CLI

Create a binary crate and add the SDK. The native build uses the default
`rustls-tls` backend; `tokio` runs the async runtime and `serde_json` builds
GraphQL variables:

```bash Terminal
cargo new --bin notes-cli
cd notes-cli

cargo add nhost
cargo add tokio --features rt-multi-thread,macros
cargo add serde_json
```

Now write a minimal `src/main.rs`. It reads your project's subdomain and region
from the environment, creates the Nhost client, and dispatches to a command. We
add real commands in the following parts; for now it just prints usage.

```rust title="src/main.rs"
use nhost::{create_client, NhostClient, Options};

type Result<T> = std::result::Result<T, Box<dyn std::error::Error>>;

#[tokio::main]
async fn main() {
    let args: Vec<String> = std::env::args().skip(1).collect();
    if args.is_empty() {
        usage();
        std::process::exit(2);
    }
    if let Err(e) = run(&args[0], &args[1..]).await {
        eprintln!("error: {e}");
        std::process::exit(1);
    }
}

async fn run(cmd: &str, args: &[String]) -> Result<()> {
    let client = make_client();
    let _ = (&client, args); // used by commands added in later parts
    match cmd {
        "help" | "-h" | "--help" => {
            usage();
            Ok(())
        }
        other => {
            usage();
            Err(format!("unknown command {other:?}").into())
        }
    }
}

// make_client builds the Nhost client. The client middleware attaches the
// access token to every request and refreshes it automatically once we sign in.
fn make_client() -> NhostClient {
    create_client(Options {
        subdomain: Some(env("NHOST_SUBDOMAIN", "local")),
        region: Some(env("NHOST_REGION", "local")),
        ..Default::default()
    })
}

fn env(key: &str, fallback: &str) -> String {
    std::env::var(key).unwrap_or_else(|_| fallback.to_string())
}

fn usage() {
    eprintln!(
        "notes-cli — a note-taking CLI built on Nhost\n\n\
         Usage:\n  notes-cli <command> [args]"
    );
}
```

Point the CLI at your project by exporting the subdomain and region you noted
above (or leave them unset to default to a local backend):

```bash Terminal
export NHOST_SUBDOMAIN=<your-subdomain>
export NHOST_REGION=<your-region>

cargo run -- help
```

You should see the usage text. The client is wired up and ready.
