Nhost Functions inject a permissive set of CORS headers by default, so calls from a browser on a different origin (for example, a frontend on `localhost:5173` calling functions at `local.functions.local.nhost.run`) work out of the box. The defaults apply identically in local development (`nhost up`) and in production.

## Default Headers

Every function response includes:

| Header                          | Value                                          |
| ------------------------------- | ---------------------------------------------- |
| `Access-Control-Allow-Origin`   | `*`                                            |
| `Access-Control-Allow-Headers`  | `origin,Accept,Authorization,Content-Type`     |

`Access-Control-Allow-Methods` and `Access-Control-Allow-Credentials` are **not** set by default. If you need them, set them yourself (see below).

:::caution
`*` allows any website to call your function. In production, restrict the origin to your actual frontend(s).
:::

## Customizing

Use the [`cors`](https://www.npmjs.com/package/cors) npm package for the cleanest setup:

```shell
npm install cors
npm install -D @types/cors
```

```ts title="./functions/api.ts"
import type { Request, Response } from 'express'
import cors from 'cors'

const corsMiddleware = cors({
  origin: ['https://myapp.com', 'http://localhost:5173'],
  methods: ['GET', 'POST', 'PUT', 'DELETE'],
})

export default (req: Request, res: Response) => {
  corsMiddleware(req, res, () => {
    res.status(200).json({ message: 'Hello from a CORS-enabled function!' })
  })
}
```

Or set headers manually:

```ts title="./functions/api.ts"
import type { Request, Response } from 'express'

export default (req: Request, res: Response) => {
  res.setHeader('Access-Control-Allow-Origin', 'https://myapp.com')
  res.status(200).json({ message: 'Hello!' })
}
```

Any header you set yourself overwrites the default for that header. Headers you don't set keep their default values.

## Credentials

If your frontend sends cookies or authorization headers with `credentials: 'include'`, you need `Access-Control-Allow-Credentials: true`. The CORS spec forbids combining that with `Access-Control-Allow-Origin: *`, so you must also override the origin to a specific value:

```ts
res.setHeader('Access-Control-Allow-Origin', 'https://myapp.com')
res.setHeader('Access-Control-Allow-Credentials', 'true')
```

## Preflight Requests

Browsers send a preflight `OPTIONS` request before the actual request when:

- The request uses a method other than GET, HEAD, or POST
- The request includes custom headers beyond the defaults above
- The request uses `Content-Type` other than `application/x-www-form-urlencoded`, `multipart/form-data`, or `text/plain`

Your function receives all HTTP methods on the same handler, including `OPTIONS`. For non-trivial cases (custom methods, dynamic origins, credentials), the `cors` package handles preflight automatically.
