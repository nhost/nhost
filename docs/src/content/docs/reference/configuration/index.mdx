This page documents every field available in your project's `nhost.toml`, generated
directly from the configuration schema. For task-oriented guidance and copy-paste
examples, see the per-product documentation; this page is the exhaustive field reference.

## Top-level structure

The root of `nhost.toml` is made up of the following sections.

| Section | Description |
|---|---|
| [`global`](#global) | Global configuration that applies to all services |
| [`hasura`](#hasura) | Configuration for hasura |
| [`graphql`](#graphql) | Advanced configuration for GraphQL |
| [`functions`](#functions) | Configuration for functions service |
| [`auth`](#auth) | Configuration for auth service |
| [`postgres`](#postgres) | Configuration for postgres service |
| [`provider`](#provider) | Configuration for third party providers like SMTP, SMS, etc. |
| [`storage`](#storage) | Configuration for storage service |
| [`ai`](#ai) | Configuration for graphite service |
| [`observability`](#observability) | Configuration for observability service |
| [`experimental`](#experimental) | Experimental configuration for unreleased services. Subject to breaking changes. |

| Field | Type | Default | Description |
|---|---|---|---|
| `compute?` | [`ResourcesCompute`](#resourcescompute) |  | CPU and memory allocation. |
| `replicas` | `uint8 & >=1 & <=10` | `1` | Number of replicas for a service |
| `autoscaler?` | [`Autoscaler`](#autoscaler) |  | Automatic replica scaling settings. |
| `networking?` | [`Networking`](#networking) \| `null` |  | Network exposure and ingress configuration. |

### RateLimit

|---|---|---|---|

	password: string
	sender:   string

	user?:   string
	port?:   #Port
	secure?: bool
	method?: "LOGIN" | "CRAM-MD5" | "PLAIN"

	user:     string
	password: string
	sender:   string

}
