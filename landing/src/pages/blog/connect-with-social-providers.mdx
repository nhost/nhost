You can now allow users to sign in to your app using multiple authentication providers by linking it to an existing user account. Users are identifiable by the same user ID regardless of the authentication provider they used to sign in. For example, a user who signed in with a password can link a GitHub account and sign in with both in the future. In a few easy steps you can enable this in your app.

## 1. Enable the social auth provider you want your users to be able to link to their account

For example in the screenshot below we enable Signin with GitHub which you can find when you click on **Sign-In Methods** under Settings.

<img
  src="/images/blog/connect-with-social-providers/enable-github-signin.png"
  alt="Enable Sign-In with GitHub"
/>

## 2. Generate the link from our SDK's

Once that's done, you just need to use our SDK to redirect the user to **hasura-auth** to communicate with the authentication provider.

The following is an example of how to do that with React.

```tsx
import React from 'react'
import { useProviderLink } from '@nhost/react'

export const Profile: React.FC = () => {
  const { github } = useProviderLink({
    connect: true,
    redirectTo: `${window.location.origin}/profile`,
  })

  return (
    <div>
      <a href={github}>Connect with GitHub</a>
    </div>
  )
}
```

Notice we've added a new paramater that you must pass to the `useProviderLink` hook to generate the right link.

```diff
const { github } = useProviderLink({
+  connect: true,
  redirectTo: `${window.location.origin}/profile`,
})
```

## 3. Verify that the provider was added to the user's account

To verify that the provider was linked to the user's account we just need to query the `user_providers` table using the custom hook `useAuthQuery` exported from our React Apollo integration package. PS. make sure to install `@nhost/react-apollo` by running `npm install @nhost/react-apollo`.

```tsx
import { useAuthQuery } from '@nhost/react-apollo'

const {
  data: { authUserProviders },
} = useAuthQuery(gql`
  query getAuthUserProviders {
    authUserProviders {
      id
      providerId
    }
  }
`)

const isGithubLinked = authUserProviders?.some(
  (item: { providerId: string }) => item.providerId === 'github',
)

console.log({
  isGithubConnected,
})
```

## 4. Putting it all together

Here's what an example page in React looks like after following the steps above. Now we just to check if the user has already connected their github account and only show the link otherwise.

```tsx
import React from 'react'
import { useProviderLink } from '@nhost/react'
import { useAuthQuery } from '@nhost/react-apollo'

export const Profile: React.FC = () => {
  const { github } = useProviderLink({
    connect: true,
    redirectTo: `${window.location.origin}/profile`,
  })

  const {
    data: { authUserProviders },
  } = useAuthQuery(gql`
    query getAuthUserProviders {
      authUserProviders {
        id
        providerId
      }
    }
  `)

  const isGithubLinked = authUserProviders?.some(
    (item: { providerId: string }) => item.providerId === 'github',
  )

  return (
    <div>
      {isGithubConnected ? (
        <span>GitHub connected</span>
      ) : (
        <a href={github}>Connect with GitHub</a>
      )}
    </div>
  )
}
```

### Wrap up

To enable additional social authentication providers like Facebook, Google, or LinkedIn, simply enable them in your app's dashboard settings. Afterwards, use our SDKs to generate the appropriate link for each provider, ensuring seamless integration. This allows for a diverse array of sign-in options, enhancing user accessibility and convenience.
