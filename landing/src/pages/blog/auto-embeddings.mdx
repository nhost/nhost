## Nhost Postgres, your Vector Database

We're in the midst of the AI revolution and efficient data processing has become crucial for applications that involve large language models and semantic search. 

Duo to their efficiency and scalability, **Vector Databases** have become a cornerstone of the AI revolution. A Vector Database is a specialized type of database that can store high-dimensional vectors allowing AI applications to efficiently search through data with a query vector and retrieve the closest matching database records. 

Nhost Postgres is now a vector database and you can use it to store vectors and embeddings to unlock a hole new set of applications around:

- search
- classification
- clustering
- recommendation
- etc

Nhost leverages **pgvector**, an extension for Postgres designed to support efficient searching and operations on high-dimensional vectors required by AI applications working and searching through complex entities like images, documents, text, or videos.

Key features of pgvector include:

- **Vector Indexing** to allow for the creation of indexes on vector columns, which can significantly speed up nearest neighbor searches
- Support for **High-Dimensional Data**
- Various Distance Metrics usually used in the vector space (L2 distance, cosine similarity, etc)
- **Ease of use**, specially for those of you already familiar with PostgreSQL

Turning your Nhost Postgres instance into a Vector Database is possible today starting with Nhost Postgres version **14.6-20230525**. You can install it with:

```psql
CREATE EXTENSION IF NOT EXISTS vector;
```

### Transforming Data into Actionable Insights

Imagine effortlessly querying your data using **natural language** or finding similar objects in your database through a similarity search. Auto-Embeddings turns this into reality, empowering your applications with limitless functionality.

#### The Superpowers

Integrated with our GraphQL API, Auto-Embeddings offers a seamless and user-friendly experience.

Once the **movies** Auto-Embedding is configured, the following queries (graphiteSearchXXX and graphiteSimilarXXX) will be available:

- **graphiteSearchMovies**
- **graphiteSearchMoviesAggregate**
- **graphiteSimilarMovies**
- **graphiteSimilarMoviesAggregate**

These queries will work similar to the standard **movies** and **moviesAggregate** queries provided by Hasura and will respect the exact **same permissions**, seamlessly.
