## Nhost Run

Nhost Run works with the same tools you already use and love: Deployments with Git, CLI, and configuration file.

Running your services is as simple as defining your service's configuration with a similar TOML file you already use. 

1. Define your service configuration

```toml
name = 'mysvc'

[image]
image = 'dummy-value'

[[ports]]
port = 3000
type = 'http'
publish = true

[resources]
replicas = 1

[resources.compute]
cpu = 62
memory = 128
```

## Benefits of Run

- Close Proximity: All your services run in close proximity.

- Seamless Scaling: Adjust your Compute Resources allocation as your apps grow, ensuring that you can support increased traffic and demand without compromising performance or user experience.

- Cost Efficiency: Allocate resources precisely **where** and **when** they are needed, avoiding over-provisioning and reducing your monthly bill. You can increase and decrease compute at any time.
