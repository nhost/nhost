## How does it work?

### Next.js Frontend

The frontend is built using Next.js together with TypeScript and TailwindCSS. GraphQL is handled by [GraphQL Code Generator and React Query](https://nhost.io/blog/how-to-use-graphql-code-generator-with-react-query) which works beautifully with TypeScript!

### Nhost Backend

The backend is (obviously) powered by Nhost, providing a Postgres database, GraphQL API, and Serverless Functions for handling Stripe webhooks and generating a [Stripe GraphQL API](https://docs.nhost.io/graphql/remote-schemas/stripe). In addition, the backend also includes [Magic Link](https://docs.nhost.io/authentication/sign-in-with-magic-link) authentication, with the ability to customize email templates.

This combination of backend elements creates a solid fundament and starting point for your next SaaS application.

### Stripe Payments

Payments are handled by Stripe, which is integrated with the backend in two ways.

GraphQL API: We're using the open source [Stripe GraphQL API](https://github.com/nhost/nhost/tree/main/integrations/stripe-graphql-js) using a Nhost Function and Hasura Remote Sceham. This way, it's easy to get data from Stripe like customer information or invoice URLs.

Stripe Wekbooks: We're using a Nhost Function to receive [Stripe webhooks](https://stripe.com/docs/webhooks) to listen to events from Stripe. This can be set up to listen to any Stripe event and is pre-configured to listen to subscriptions created and deleted.

Stripe Customer Portal: We're using the [Stripe Customer Portal](https://stripe.com/docs/customer-management) to let the user manage their subscriptions. This way, we save ourselves tedious time to develop complex UI and payment management.
