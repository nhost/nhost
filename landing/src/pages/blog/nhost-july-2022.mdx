## Dedicated Postgres Instances per Nhost Project (Preview)

If you didn't know, we're currently hosting Postgres on AWS RDS in a shared environment. This didn't scale well so we're moving to provide dedicated Postgres instances per Nhost project. Most of the work is completed and we're now testing the new architecture.

If you want to be part of the tests, feel free to reach out to us at [support@nhost.io](mailto:support@nhost.io) or via our [Discord server](/discord).

We expect to have a dedicated Postgres instance per Nhost project in the next few weeks.

This is the timeline:

1. New Nhost projects get a dedicated Postgres instance.
2. Existing Nhost projects can manually migrate to a dedicated Postgres instance (one-click migration)
3. We will automatically migrate all remaining Nhost projects to a dedicated Postgres instance.
4. We will shut down our AWS RDS instances.

Providing individual Postgres instances per Nhost project is one of the [most requested features](https://github.com/nhost/nhost/issues/113). This will give you full root access to Postres and enable you to do things like installing Postgres extensions.

## Data Editor: Edit and Delete Data

Last week we launched the Data Editor for the database with read-only access. This month, we launched the ability to edit and delete existing data.

The editor works as follows:

- One-click selects a cell
- Double click starts editing in-place (boolean columns use a drop-down). You can also hit Enter to start editing if the cell is selected.
- To commit changes either use Enter or click outside of the cell.
- To ignore changes use Esc.

Here's a short video of how to edit data. Notice how you can edit and delete data in the same way you would in popular services like Airtable, Excel, or Notion. We're paying a lot of attention to this Data Editor to make the usability and accessibility world-class!

## Apple Sign-In

![Sign-In with Apple](/images/blog/nhost-july-2022/sign-in-with-apple.png)

You can now sign in users with Apple. This is particularly important for users who are building apps for the Apple App Store. Once you have configured your Nhost project to use Apple Sign-In it's simple to sign in users with Apple:

```js
nhost.auth.signIn({
  provider: 'apple',
})
```

Check our docs: [How to sign in users with Apple](https://docs.nhost.io/products/auth/social/sign-in-apple).
